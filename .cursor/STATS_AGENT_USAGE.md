# Stats Agent - Использование

## Обзор

Stats Agent собирает статистику по задачам всех агентов и составляет таблицы с метриками производительности. Агент оптимизирован для экономии квоты GitHub API.

## Быстрый старт

### Базовая статистика

```
@stats Покажи статистику по всем агентам
```

Агент:
1. Использует один запрос поиска для всех Issues с метками агентов
2. Собирает статистику: всего задач, открыто, закрыто, в работе, возвращено
3. Составляет таблицу с метриками
4. Вычисляет прогресс для каждого агента

### Детальная статистика

```
@stats Собери детальную статистику по всем агентам с разбивкой по этапам
```

### Статистика конкретного агента

```
@stats Покажи статистику для Backend Developer
```

## Оптимизация запросов

Stats Agent использует оптимизированные запросы к GitHub API:

### ✅ Один запрос для всех Issues

```javascript
// Один запрос вместо множественных
const result = await mcp_github_search_issues({
  query: 'is:issue label:agent:*',
  perPage: 100
});
```

### ✅ Кэширование результатов

- Кэш работает в рамках одной сессии
- TTL: 2 минуты для результатов поиска
- Повторные запросы используют кэш

### ✅ Задержки между запросами

- 500ms между запросами при детальной статистике
- Минимизирует риск rate limit

## Примеры использования

### Ежедневная статистика

```
@stats Собери ежедневную статистику и покажи изменения за последние 7 дней
```

### Загрузка агентов

```
@stats Покажи загрузку агентов - у кого больше всего задач в работе
```

### Статистика по приоритетам

```
@stats Покажи статистику по приоритетам задач
```

### Статистика возвращенных задач

```
@stats Покажи статистику по возвращенным задачам - какие агенты чаще возвращают задачи
```

## Метрики

Stats Agent собирает следующие метрики:

1. **Всего задач** - общее количество Issues с меткой агента
2. **Открыто** - количество открытых Issues
3. **Закрыто** - количество закрытых Issues
4. **В работе** - открытые Issues без метки `returned`
5. **Возвращено** - открытые Issues с меткой `returned`
6. **Прогресс** - процент закрытых задач (закрыто / всего * 100%)

## Ограничения

- Кэш работает только в рамках одной сессии
- Статистика собирается на момент запроса
- Для исторических данных нужны дополнительные запросы
