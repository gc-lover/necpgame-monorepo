# QA: Найти мои задачи

Найти все открытые задачи для QA через MCP GitHub с оптимизацией запросов.

## Инструкции

1. **Используй поиск с кэшированием (ОБЯЗАТЕЛЬНО):**

   ```javascript
   const query = 'is:issue is:open label:agent:qa';
   const result = await mcp_github_search_issues({
     query: query,
     perPage: 100
   });
   ```

2. **Проверь готовность функционала:**
   - Должен быть готовый бекенд (от Backend Developer)
   - Должен быть готовый клиент (от UE5 Developer, если применимо)
   - **Проверь, что это НЕ контентный квест (YAML файл)** - QA тестирует только готовый функционал

3. **Отфильтруй и покажи список задач:**
   - Номер Issue
   - Название
   - Есть ли готовый функционал
   - Приоритет

4. **Спроси пользователя, с какой задачей работать**

## Оптимизация

- **КРИТИЧЕСКИ ВАЖНО:** Используй `mcp_github_search_issues` вместо множественных `mcp_github_issue_read`
- Используй кэширование для повторных запросов
- Добавляй задержки между запросами (500ms)
