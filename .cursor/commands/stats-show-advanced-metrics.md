# Stats: Show Advanced Metrics

Расширенные метрики конвейера для отслеживания производительности.

## 📊 Метрики

### 1. Время на этапах

**Для каждого этапа собирать:**
- Среднее время
- Медиана (P50)
- P95 (95% задач завершаются за это время)
- Минимум/Максимум

```javascript
// Пример запроса
mcp_github_list_project_items({
  owner_type: 'user',
  owner: 'gc-lover',
  project_number: 1,
  query: 'Status:"Backend - In Progress" OR Status:"Backend - Review" OR Status:"Done"',
  fields: ['updated_at', 'created_at']  // Для расчета времени
});

// Для каждой задачи
// - Найти когда задача пришла на Backend (из comments или events)
// - Найти когда задача ушла с Backend
// - Рассчитать time_on_stage = departure_time - arrival_time
// - Собрать статистику
```

### Пример вывода:

```markdown
## ⏱️ Время на этапах (последние 30 дней)

| Этап | SLA | Среднее | Медиана | P95 | Min | Max |
|------|-----|---------|---------|-----|-----|-----|
| Idea Writer | 1д | 0.8д | 0.5д | 1.5д | 0.2д | 3д |
| Architect | 2д | 1.5д | 1д | 3д | 0.5д | 5д |
| Backend | 3д | 2.5д | 2д | 5д | 1д | 8д |
| QA | 1д | 0.8д | 0.5д | 2д | 0.3д | 3д |

**Узкие места:**
🔴 Backend: Среднее 2.5д, но P95 = 5д (некоторые задачи очень долгие)
🟡 Architect: P95 = 3д (иногда превышает SLA)
```

### 2. Качество передачи (Return Rate)

**Для каждого этапа считать:**
- Сколько задач передано дальше
- Сколько задач возвращено
- % возвратов

```javascript
// Для Backend
// Передано: Backend → Network (успешно)
// Возвращено: Network → Backend (returned)
// Return rate = возвращено / передано * 100%
```

### Пример вывода:

```markdown
## 📈 Качество передачи (последние 30 дней)

| Этап → Следующий | Передано | Возвращено | Return Rate | Цель |
|------------------|----------|------------|-------------|------|
| Backend → Network | 45 | 7 | 15.6% | <15% |
| Network → Security | 38 | 3 | 7.9% | <15% |
| Security → DevOps | 35 | 2 | 5.7% | <15% |
| Backend → QA | 10 | 3 | 30% | <15% |

**Проблемы:**
🔴 Backend → QA: 30% return rate (выше цели!)
  - Причины: недостаточное тестирование перед передачей
  - Action: Backend улучшить валидацию

🟡 Backend → Network: 15.6% (на границе)
  - Причины: OpenAPI spec issues
  - Action: Проверить процесс между API Designer и Backend
```

### 4. SLA Compliance

**Для каждого агента:**
- Сколько задач завершено в SLA
- Сколько задач превысили SLA
- % compliance

```javascript
// Для Backend (SLA = 3 дня)
// - Задачи завершенные за ≤3 дня: X
// - Задачи завершенные за >3 дня: Y
// - Compliance = X / (X + Y) * 100%
```

```markdown

  - Action: Проверить причины задержек
```

## 🔧 Реализация

### Алгоритм сбора метрик:

```javascript
// 1. Получить все Issues за период
const issues = await mcp_github_list_project_items({
  owner_type: 'user',
  owner: 'gc-lover',
  project_number: 1,
  query: 'updated:>@today-30d'  // Последние 30 дней
});

// 2. Для каждой задачи:
// - Получить историю изменений статусов (из comments/events)
// - Рассчитать время на каждом этапе
// - Определить были ли возвраты
// - Проверить SLA compliance

// 3. Агрегировать статистику
// - По этапам
// - По агентам
// - По периодам

// 4. Сформировать отчет в Markdown
```

### Кэширование:

```markdown
**ВАЖНО:** Сбор метрик тяжелый (много API запросов)

**Решение:**
- Кэшировать результаты (TTL: 4 часа)
- Обновлять по расписанию (cron: каждые 4 часа)
- Хранить в Issue или Wiki как отчет
```

### Пример использования:

```bash
# Вручную запросить метрики
@stats Покажи расширенные метрики конвейера

# Или через команду
/stats-show-advanced-metrics
```
