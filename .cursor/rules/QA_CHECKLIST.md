# QA Agent - Чеклист передачи задач

## ✅ При завершении тестирования ОБЯЗАТЕЛЬНО:

### 1. Оценка результатов
- [ ] Все тесты выполнены
- [ ] Отчет создан
- [ ] Баги задокументированы (если есть)

### 2. Обновление меток

**Если есть баги:**
- [ ] Удалить `agent:qa`
- [ ] Удалить `stage:testing`
- [ ] Добавить `agent:backend` (или `agent:ue5`)
- [ ] Добавить соответствующий `stage:backend-dev` (или `stage:client-dev`)

**Если всё готово:**
- [ ] Удалить `agent:qa`
- [ ] Удалить `stage:testing`
- [ ] Добавить `agent:release`
- [ ] Добавить `stage:release`

### 3. Обновление Project (если готово)
- [ ] `Development Stage` = `release`

### 4. Комментарий в Issue
- [ ] Добавлен комментарий с результатами
- [ ] Указаны найденные баги (если есть)
- [ ] Указан следующий агент

### 5. Коммиты (если есть файлы)
- [ ] Префикс `[qa]`
- [ ] Указан номер Issue

## ❌ НИКОГДА не делать:
- ❌ Исправлять баги (только документировать)
- ❌ Менять код без явного запроса
- ❌ Забывать обновлять метки
- ❌ Забывать передавать задачу следующему агенту
