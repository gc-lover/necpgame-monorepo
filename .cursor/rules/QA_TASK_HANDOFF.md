# QA Agent - Процесс передачи задач

## OK Чеклист перед передачей задачи

### 1. Оценка результатов
- [ ] Все тесты выполнены
- [ ] Отчет создан
- [ ] Баги задокументированы (если есть)

### 2. Обновление меток

**Если есть баги:**
- [ ] Удалить `agent:qa`
- [ ] Удалить `stage:testing`
- [ ] Добавить `agent:backend` (или `agent:ue5`)
- [ ] Добавить соответствующий `stage:backend-dev` (или `stage:client-dev`)

**Если всё готово:**
- [ ] Удалить `agent:qa`
- [ ] Удалить `stage:testing`
- [ ] Добавить `agent:release`
- [ ] Добавить `stage:release`

### 3. Обновление Project (если готово)
- [ ] `Development Stage` = `release`

### 4. Комментарий в Issue
- [ ] Добавлен комментарий с результатами
- [ ] Указаны найденные баги (если есть)
- [ ] Указан следующий агент

### 5. Коммиты (если есть файлы)
- [ ] Префикс `[qa]`
- [ ] Указан номер Issue

## ❌ НИКОГДА не делать:
- ❌ Исправлять баги (только документировать)
- ❌ Менять код без явного запроса
- ❌ Забывать обновлять метки
- ❌ Забывать передавать задачу следующему агенту

## Детальный процесс передачи задач

### При завершении тестирования

### Шаг 1: Оценка результатов

**Если найдены баги:**
- Создать Issues для каждого бага
- Документировать все найденные проблемы
- Вернуть задачу разработчику

**Если тестирование успешно:**
- Все тесты пройдены
- Отчет создан
- Передать задачу агенту release

### Шаг 2: Обновление меток

#### Если есть баги (возврат разработчику):

```javascript
// Удалить метки QA
- agent:qa ❌
- stage:testing ❌

// Добавить метки разработчика
+ agent:backend (или agent:ue5)
+ stage:backend-dev (или stage:client-dev)
```

#### Если всё готово (передача release):

```javascript
// Удалить метки QA
- agent:qa ❌
- stage:testing ❌

// Добавить метки release
+ agent:release OK
+ stage:release OK
```

### Шаг 3: Обновление Project статуса

Если всё готово:
- `Development Stage` = `release`

### Шаг 4: Комментарий в Issue

#### Шаблон для багов:

```markdown
❌ **Тестирование выявило баги**

**Статус:** Задача возвращена разработчику для исправления

**Найденные баги:**
- Issue #XXX: [Краткое описание бага]
- Issue #YYY: [Краткое описание бага]

**Метки обновлены:**
- Удалено: `agent:qa`, `stage:testing`
- Добавлено: `agent:backend`, `stage:backend-dev`

**Следующие шаги:**
После исправления багов задача будет возвращена на тестирование.
```

#### Шаблон для успешного тестирования:

```markdown
OK **Тестирование завершено, готово к релизу**

**Статус:** Все тесты пройдены успешно

**Результаты тестирования:**
- OK Все функциональные тесты пройдены
- OK Интеграционные тесты пройдены
- OK API тесты пройдены
- OK Нагрузочные тесты пройдены
- OK Edge cases обработаны корректно
- OK Отчет о тестировании создан

**Метки обновлены:**
- Удалено: `agent:qa`, `stage:testing`
- Добавлено: `agent:release`, `stage:release`

**Статус Project обновлен:**
- `Development Stage` = `release`

Задача передана агенту @agent:release
```

### Шаг 5: Коммиты

После создания тестов и отчетов:

```bash
git commit -m "[qa] test: добавить тест-кейсы для character-service-go

- Добавлены функциональные тесты
- Добавлены интеграционные тесты
- Создан отчет о тестировании

Связанные Issue: #143"
```

## Важно

- **НЕ исправляй баги** - только документируй
- **НЕ меняй код** без явного запроса
- **ВСЕГДА передавай задачу** следующему агенту после завершения
- **ВСЕГДА обновляй метки** при передаче
