# Правила Cursor в формате MDC

## Что такое MDC?

**MDC (Markdown with Context)** - это формат файлов для создания правил в Cursor IDE. Файлы с расширением `.mdc` содержат метаданные и инструкции, которые помогают AI-агенту понимать и следовать определённым стандартам и процессам в проекте.

## Структура MDC файла

MDC файлы используют frontmatter (YAML) в начале для метаданных:

```yaml
---
description: "Описание правила"
alwaysApply: true/false
globs: ["**/*.cpp", "**/*.h"]
priority: 1-10
tags: ["tag1", "tag2"]
---
```

### Параметры метаданных

- **description**: Описание правила для AI-агента
- **alwaysApply**: Применять ли правило всегда (true) или только при необходимости (false)
- **globs**: Паттерны файлов, к которым применяется правило
- **priority**: Приоритет правила (1 - высший, 10 - низший)
- **tags**: Теги для категоризации правил

## Созданные правила

### 1. SOLID.mdc
- **Описание**: SOLID принципы для всех файлов проекта
- **Приоритет**: 1 (высший)
- **Применение**: Всегда для всех C++/C#/Java файлов
- **Содержание**: 
  - Все 5 принципов SOLID
  - Правила для UE5/C++
  - Примеры нарушений и правильных решений
  - Чеклист проверки

### 2. UE5-CPP-STYLE.mdc
- **Описание**: Стиль кода для UE5/C++
- **Приоритет**: 2
- **Применение**: Всегда для всех C++ файлов
- **Содержание**:
  - Именование классов и функций
  - Структура файлов
  - UE5 специфичные правила
  - Правила работы с памятью

### 3. CODE-REVIEW-CHECKLIST.mdc
- **Описание**: Чеклист для проверки кода
- **Приоритет**: 3
- **Применение**: По необходимости (alwaysApply: false)
- **Содержание**:
  - Чеклист SOLID принципов
  - Проверка размера и структуры
  - Проверка качества кода
  - UE5 специфичные проверки

## Расположение правил

Правила находятся в `.cursor/rules/`:
```
.cursor/
└── rules/
    ├── SOLID.mdc
    ├── UE5-CPP-STYLE.mdc
    └── CODE-REVIEW-CHECKLIST.mdc
```

## Как работают правила

1. **Автоматическое применение**: Правила с `alwaysApply: true` применяются автоматически при работе с файлами, соответствующими `globs`
2. **Приоритет**: Правила с меньшим приоритетом применяются первыми
3. **Контекст**: AI-агент использует правила как контекст для понимания требований проекта

## Преимущества MDC формата

1. **Метаданные**: Позволяют управлять применением правил
2. **Версионный контроль**: Правила в репозитории, команда работает по одним стандартам
3. **Гибкость**: Можно создавать специфичные правила для разных типов файлов
4. **Автоматизация**: AI-агент автоматически следует правилам

## Миграция с .md на .mdc

Старый формат `.cursor/rules/SOLID.md` был удалён и заменён на `.cursor/rules/SOLID.mdc` с метаданными.
