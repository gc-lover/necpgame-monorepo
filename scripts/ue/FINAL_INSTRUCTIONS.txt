========================================
ИНСТРУКЦИЯ ПО СОЗДАНИЮ BLUEPRINT ФАЙЛОВ
========================================

ПРОБЛЕМА:
Автоматическое создание Blueprint'ов через командную строку не работает,
так как C++ классы не загружаются в Python API до полной загрузки редактора.

РЕШЕНИЕ - ВЫПОЛНИТЕ ВРУЧНУЮ В РЕДАКТОРЕ:
========================================

1. Откройте Unreal Editor с проектом necpg.uproject
   (если уже открыт - используйте его)

2. Дождитесь полной загрузки проекта

3. Нажмите `` ` `` (обратная кавычка) для открытия консоли

4. Выполните один из скриптов:

   Вариант A (рекомендуется):
   py Scripts/create_blueprints_manual_setup.py
   
   Это создаст Blueprint'ы с базовыми классами, затем:
   - Откройте каждый Blueprint
   - В Class Settings установите Parent Class:
     * WBP_TechDemoHUD -> TechDemoHUDWidget
     * BP_TechDemoHUD -> TechDemoHUD  
     * BP_TechDemoGameMode -> MyGameMode

   Вариант B:
   py Scripts/create_hud_blueprints_simple.py
   
   Попытается создать с правильными классами (может не сработать)

5. После создания Blueprint'ов:
   - Откройте WBP_TechDemoHUD
   - Добавьте 4 Text виджета с именами:
     * FPSText
     * PingText
     * ConnectionStatusText
     * PlayerCountText
   
   - Откройте BP_TechDemoHUD
   - Установите HUD Widget Class = WBP_TechDemoHUD
   
   - Откройте BP_TechDemoGameMode
   - Установите HUD Class = BP_TechDemoHUD

========================================
АЛЬТЕРНАТИВА - СОЗДАНИЕ ВРУЧНУЮ:
========================================

1. Content Browser -> Right Click -> User Interface -> Widget Blueprint
   Назвать: WBP_TechDemoHUD
   Parent Class: TechDemoHUDWidget

2. Content Browser -> Right Click -> Blueprint Class
   Назвать: BP_TechDemoHUD
   Parent Class: TechDemoHUD

3. Content Browser -> Right Click -> Blueprint Class
   Назвать: BP_TechDemoGameMode
   Parent Class: MyGameMode
