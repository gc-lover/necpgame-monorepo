========================================
ПРОБЛЕМА: Редактор закрывается сразу после открытия
========================================

ПРИЧИНА:
Когда используется параметр -ExecutePythonScript, редактор автоматически
закрывается после выполнения Python скрипта. Это нормальное поведение
для автоматизации, но не подходит для ручной работы.

РЕШЕНИЕ:
========================================

ВАРИАНТ 1: Простой запуск (рекомендуется)
----------------------------------------
Используйте скрипт: scripts\ue\launch_editor_simple.cmd

Этот скрипт запускает редактор БЕЗ автоматического выполнения скриптов.
Редактор останется открытым для ручной работы.

После открытия редактора:
1. Нажмите `` ` `` (обратная кавычка) для открытия консоли
2. Выполните: py Scripts/create_blueprints_manual_setup.py
3. Редактор останется открытым

========================================

ВАРИАНТ 2: Двойной клик по .uproject
----------------------------------------
Просто откройте файл:
C:\NECPGAME\client\UE5\necpg\necpg.uproject

Двойным кликом в проводнике Windows.

========================================

ВАРИАНТ 3: Через Epic Games Launcher
----------------------------------------
1. Откройте Epic Games Launcher
2. Перейдите в Unreal Engine
3. Найдите проект necpg
4. Нажмите "Открыть"

========================================

ВАРИАНТ 4: Прямая команда (PowerShell)
----------------------------------------
& "C:\Program Files\Epic Games\UE_5.6\Engine\Binaries\Win64\UnrealEditor.exe" "C:\NECPGAME\client\UE5\necpg\necpg.uproject"

========================================

ПОЧЕМУ СКРИПТЫ С -ExecutePythonScript ЗАКРЫВАЮТ РЕДАКТОР:
--------------------------------------------------------
Это стандартное поведение UE5 для автоматизации:
- Скрипт выполняется
- Редактор автоматически закрывается
- Это нужно для CI/CD и автоматизации

Для ручной работы используйте launch_editor_simple.cmd или
запускайте редактор обычным способом.
