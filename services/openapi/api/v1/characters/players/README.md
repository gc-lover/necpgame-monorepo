## Player Character Lifecycle API

- `players.yaml` — основной контракт character-service для управления жизненным циклом персонажей (создание, удаление, восстановление, переключение, слоты, аудит).
- `players-models.yaml` — базовые схемы: профиль игрока, резюме персонажа, состояния слотов, очередь восстановления, активности.
- `players-models-operations.yaml` — модели запросов/ответов, Kafka события `characters.lifecycle.*`, структуры очередей и платежей.

Каждый файл ≤ 400 строк, что соответствует требованиям по размеру спецификаций.
