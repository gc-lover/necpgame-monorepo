## Описание

API для управления квестовой системой в текстовой версии NECPGAME.

Реализует:
- Просмотр доступных квестов
- Активные квесты персонажа
- Принятие квестов от NPC
- Отслеживание прогресса выполнения
- Завершение квестов и получение наград
- Отказ от квестов

## API Файлы

### quests.yaml
**Описание:** Основной API для работы с квестами.

**Endpoints:**
- `GET /quests` - Список доступных квестов
- `GET /quests/active` - Активные квесты персонажа
- `GET /quests/{questId}` - Детали квеста
- `POST /quests/accept` - Принять квест
- `POST /quests/complete` - Завершить квест
- `POST /quests/abandon` - Отказаться от квеста
- `GET /quests/{questId}/objectives` - Цели квеста

**Модели:**
- `Quest` - Квест
- `QuestObjective` - Цель квеста
- `QuestRewards` - Награды за квест
- `QuestProgress` - Прогресс выполнения
- `AcceptQuestRequest` - Запрос принятия квеста
- `CompleteQuestRequest` - Запрос завершения квеста

## Связанные API

- `api/v1/game/initial-state.yaml` - Начальные квесты
- `api/v1/npcs/` - NPC дают квесты
- `api/v1/locations/` - Квесты связаны с локациями

---

## Примечания

- Квесты имеют типы: main, side, contract, daily
- Квесты имеют требования (уровень, репутация, предыдущие квесты)
- Квесты дают награды (опыт, деньги, предметы, репутация)
- Квесты могут иметь несколько этапов/целей
