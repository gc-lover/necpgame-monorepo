## Описание

API для системы локаций и перемещения в текстовой версии NECPGAME.

Реализует:
- Просмотр доступных локаций
- Получение детальной информации о локации
- Текущую локацию персонажа
- Перемещение между локациями
- Доступные действия в локации
- Связанные локации

## API Файлы

### locations.yaml
**Описание:** Основной API для работы с локациями и перемещением.

**Endpoints:**
- `GET /locations` - Список всех доступных локаций
- `GET /locations/{locationId}` - Детальная информация о локации
- `GET /locations/current` - Текущая локация персонажа
- `POST /locations/travel` - Перемещение в другую локацию
- `GET /locations/{locationId}/actions` - Доступные действия в локации
- `GET /locations/{locationId}/connected` - Связанные локации

**Модели:**
- `GameLocation` - Локация
- `LocationDetails` - Детальная информация о локации
- `LocationAction` - Действие в локации
- `TravelRequest` - Запрос перемещения
- `TravelResponse` - Ответ перемещения
- `ConnectedLocation` - Связанная локация

---

## Связанные API

- `api/v1/game/start.yaml` - Использует модель GameLocation
- `api/v1/game/initial-state.yaml` - Начальная локация персонажа
- `api/v1/npcs/` - NPC связаны с локациями
- `api/v1/quests/` - Квесты связаны с локациями

---

## Примечания

- Все endpoints требуют аутентификации (Bearer Token)
- Данные локаций хранятся в PostgreSQL
- Перемещение может требовать энергию и время
- Локации имеют уровень опасности и требования
