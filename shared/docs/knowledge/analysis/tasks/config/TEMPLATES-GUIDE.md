## 1. Общие правила

- Каждый документ ≤ 500 строк. При превышении создаём последовательности `_0001`, `_0002`, …
- Перед серийным созданием материалов обновляем соответствующий шаблон.
- Метаданные в начале документа: `Статус`, `Приоритет`, `api-readiness`, `api-readiness-notes`, `Версия`, `Обновлено`.
- Указываем целевой микросервис, каталог OpenAPI и модуль фронтенда (если документ технический).

## 4. Создание новых шаблонов

1. Определите повторяющийся формат (≥ 3 документов).
2. Согласуйте структуру с GLOBAL-RULES и профильными агентами.
3. Создайте шаблон, добавьте описание в эту таблицу.
4. Укажите версию и дату обновления.
5. Сообщите команде о появлении шаблона (через current-status).

---

## 5. Поддержка актуальности

- При обновлении шаблона фиксируйте изменения в `CURRENT-WORK/current-status.md`.
- Проверяйте, что ссылки в шаблонах ведут на актуальные документы.
- Используйте `STATUSES-GUIDE.md` для единообразия статусов.
