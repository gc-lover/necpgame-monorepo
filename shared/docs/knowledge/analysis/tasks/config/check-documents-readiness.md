# Краткая инструкция: проверка готовности документов к API

1. Получи актуальное время (`powershell -Command "Get-Date -Format 'yyyy-MM-dd HH:mm'"`).
2. Убедись, что документ ≤ 500 строк и заполнены метаданные (статус, приоритет, версия, api-readiness).
3. Пройди чеклист `ЧЕКБРЕЙН.MD`:
   - Полнота данных, конкретика, отсутствие блокирующих TODO.
   - Указаны микросервис, каталог `api/v1/...`, фронтенд модуль.
4. Определи статус готовности (ready/needs-work/blocked/in-review/not-applicable) по `STATUSES-GUIDE.md`.
5. Обнови `readiness-tracker.yaml` (добавь запись или измени существующую).
6. Синхронизируй `TODO.md`, `current-status.md`, `open-questions.md` при необходимости.
7. Сообщи Brain Manager о документах со статусом `ready` для постановки задач ДУАПИТАСК.
8. После партии изменений зафиксируй изменения вручную через git (`git add`, `git commit`, `git push`) или MCP.
