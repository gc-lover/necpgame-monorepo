# ЧЕКБРЕЙН — чеклист проверки готовности документов

**Версия:** 1.0.0  
**Обновлено:** 2025-11-08 23:49  
**Ответственный:** Brain Manager

---

## 1. Перед проверкой
- [ ] Получено фактическое время (`powershell Get-Date -Format 'yyyy-MM-dd HH:mm'`).
- [ ] Документ ≤ 500 строк, структура соответствует шаблону.
- [ ] Метаданные заполнены: статус, приоритет, версия, api-readiness, примечания.
- [ ] Существуют ссылки на связанные документы и API каталоги (если технический файл).

## 2. Контент
- [ ] Описаны ключевые механики/процессы без критичных пробелов.
- [ ] Уточнены бизнес-правила, ограничения, сценарии ошибок.
- [ ] Нет блокирующих TODO или пометок о неизвестных данных.
- [ ] Примеры/кейсы отражают реальный игровой процесс.

## 3. Готовность к API
- [ ] Определён целевой микросервис (см. GLOBAL-RULES.md).
- [ ] Указан каталог OpenAPI `api/v1/...` и фронтенд модуль (если применимо).
- [ ] Структура сущностей и событий согласуется с существующими системами.
- [ ] Документ не противоречит другим утвержденным материалам.

## 4. Обновление трекеров
- [ ] Внесена запись в `readiness-tracker.yaml` (статус + приоритет + время проверки).
- [ ] Документ содержит секцию `API Tasks` (если статус `ready`).
- [ ] `TODO.md` и `current-status.md` отражают новое состояние.
- [ ] При необходимости добавлены вопросы в `open-questions.md`.

## 5. Завершение
- [ ] Проверка зафиксирована в комментарии/истории документа.
- [ ] Изменения сохранены и подготовлены к коммиту.
- [ ] Запланировано напоминание ДУАПИТАСК о готовом документе.
- [ ] После серии проверок изменения зафиксированы вручную через git (`git add`, `git commit`, `git push`) или MCP.
