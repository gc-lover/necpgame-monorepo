## Общее описание

Раздел для обсуждения и проработки сценариев, квестов, лора игры, лора NPC, лора событий и прочей литературной части проекта NECPGAME.

**Цель:** Создать централизованное место для разработки всех литературных элементов игры, включая сценарии, квесты, истории персонажей, события и лор.

## Архитектурный контекст

- Повествовательные системы реализуются поверх микросервисного backend’а (narrative-, world-, social-service). Монолитный backend более не используется.
- Production API доступен только через `https://api.necp.game/v1` (HTTP) и `wss://api.necp.game/v1` (WebSocket); прямые порты применяются исключительно в dev-среде.
- При подготовке OpenAPI спецификаций для нарративных функций обязательно заполняйте `info.x-microservice` с указанием целевого сервиса.

## Структура раздела

### scenarios/ - Сценарии
Детальные сценарии для различных игровых ситуаций:
- Сценарии начала игры
- Сценарии событий
- Сценарии взаимодействий
- Сценарии ветвления сюжета

### quests/ - Квесты
Детальные описания квестов:
- Основные квесты (main quests)
- Побочные квесты (side quests)
- Квесты фракций
- Квесты событий
- Квесты игроков

### lore/ - Лор игры
Лор и история игрового мира:
- История вселенной
- История городов
- История фракций
- История событий
- Мифология и легенды

### npc-lore/ - Лор NPC
Детальные описания NPC и их историй:
- Важные NPC
- История NPC
- Отношения NPC
- Диалоги NPC
- Развитие NPC

### events-lore/ - Лор событий
Лор и описание игровых событий:
- Мировые события
- События фракций
- События локаций
- Динамические события
- Исторические события

### stories/ - Истории и повествования
Литературные истории и повествования:
- Истории персонажей
- Истории фракций
- Истории локаций
- Короткие рассказы
- Эпические истории

### dialogues/ - Диалоги
Диалоги и разговоры:
- Диалоги NPC
- Диалоги квестов
- Диалоги событий
- Ветвление диалогов
- Романтические диалоги

## Правила работы

1. **Структурированность:** Все документы должны быть структурированы и понятны
2. **Версионирование:** Используйте версионирование документов при необходимости
3. **Связность:** Связывайте документы между собой через ссылки
4. **Детализация:** Детализируйте сценарии, квесты и лор для полноты картины
5. **Согласованность:** Следите за согласованностью лора с общим сеттингом
