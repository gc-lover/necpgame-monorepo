Этот раздел содержит базовые концепции, видение проекта и философию дизайна игры NECPGAME.

## Структура

**Основные документы:**
- `vision.md` - Видение проекта и долгосрочные цели
- `design-philosophy.md` - Философия дизайна игры
- `core-pillars.md` - Основные столпы игры

**Расширенные документы:**
- `vision-unique-features.md` - Уникальные особенности и конкурентные преимущества
- `design-philosophy-ui-ux.md` - UI/UX принципы и визуальный дизайн
- `core-pillars-implementation.md` - Детализация реализации столпов и метрики успеха

## Архитектурный контекст

- Backend проекта полностью микросервисный; монолитные сценарии больше не поддерживаются.
- Production-доступ ко всем API осуществляется только через `https://api.necp.game/v1` (HTTP) и `wss://api.necp.game/v1` (WebSocket).
- Все OpenAPI спецификации обязаны содержать корректный блок `info.x-microservice` с указанием сервиса-владельца домена.

## Статусы документов

- `draft` - черновик
- `review` - на проверке
- `approved` - утвержден
- `deprecated` - устарел
